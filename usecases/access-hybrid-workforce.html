Securing the Hybrid Workforce

The same employee works from the office on Monday, home on Tuesday, a customer site on Wednesday and a hotel on Thursday — but legacy VPN concentrators backhaul their traffic, collapse under load and only apply security at HQ. This use case shows one consistent, optimised and secure access experience for every user, everywhere: universal ZTNA in place of the VPN estate.

Business objective

Why this matters

Hybrid work is no longer an emergency arrangement — it is how the business runs. Sales engineers on customer sites, developers at home, executives in departure lounges: the network they get should not depend on which chair they happen to be sitting in.

The objective: every user gets the same identity, the same policy, the same protection and the same performance — in the office, at home, or on the road — without IT running a parallel remote-access estate to deliver it.

The problem

Where VPN concentrators fall down

Legacy remote access was designed for the exception — the occasional traveller — not for a workforce that is remote half the week. Two structural problems follow:

Backhaul & capacity

  • All remote traffic hairpins through a concentrator at HQ — even traffic destined for M365 or SaaS
  • Appliances sized for a fraction of the workforce collapse when everyone connects at once
  • Users far from HQ pay the round-trip latency on every packet
  • Performance pain trains users to disconnect the VPN — and lose protection entirely

Split-brain security

  • The full inspection stack lives at HQ — remote users get whatever the concentrator bolts on
  • Two policy sets to maintain: office firewall rules and VPN ACLs, drifting apart over time
  • Split tunnels send internet traffic straight out of the home router, uninspected
  • Little or no visibility of what remote users experience — until the ticket arrives
Ask yourself

“Does a user working from home today get exactly the same security policy and inspection as a user at a desk in the office — and if not, which of the two is the real policy?”

The Cato solution

One front door, wherever the user is

The Cato Client (Windows, macOS, iOS, Android, Linux) connects each user to the nearest of 80+ PoPs, where the full single-pass security stack — ZTNA, FWaaS, SWG, IPS, anti-malware, TLS inspection — is applied to every flow. Policy is identity-driven via the IdP (MFA/SSO on sign-in), so the same rules follow the user to any location. The client is always-on: there is nothing to toggle, so there is no unprotected state to fall back into.

When that same user badges into the office, nothing changes. The client detects it is behind a Cato Socket and switches to office mode — traffic rides the site socket instead of a client tunnel, and hits exactly the same policy. There is no office/remote split-brain because there is only one rule base.
