What this shows

The journey, phase by phase

Every Zscaler replacement runs into the same fear: pull users off ZIA and internet security goes dark. It never has to. Cato moves the WAN first, bridges internet traffic back to ZIA over interim IPsec, and only flips web policy to the Cato PoP once it is converted and proven. Throughout the transition ZCC and the Cato Client co-exist, so protection never has a gap.

Phase by phase

What happens each phase

The same five steps, in words. The right-hand column is the answer to the objection that stalls every SSE migration — where the user’s internet traffic is protected at each moment.

PhaseWhat changesUser internet path during this phase
0 · On Zscaler Starting state — site on GRE/IPsec to ZIA, ZCC on every endpoint. No Cato yet. Zscaler ZIA (GRE/IPsec at the site, ZCC when remote)
1 · Interim IPsec Cato Socket takes the branch WAN onto the nearest PoP; interim IKEv2 IPsec bridges internet from the PoP to ZIA. Zscaler ZIA — now reached via the Cato PoP over interim IPsec
2 · Dual-agent cohort Pilot users get the Cato Client with ZCC nested inside it; WAN on Cato, cohort by cohort. Zscaler ZIA — the ZCC tunnel rides inside the Cato Client / socket tunnel
3 · Policy cutover Converted SWG/CASB/TLSi policy goes live at the PoP; ZIA bridge drops to standby. Cato PoP — direct internet breakout, ZIA kept on standby as a fallback
4 · ZIA / ZCC retired ZCC uninstalled, ZIA decommissioned. Site and users fully on Cato. Cato PoP — one client, one platform for WAN and internet
Why there is never a gap

The dual-agent bridge

The mechanic that removes the risk is co-existence. Through the transition (phases 1–3) ZCC and the Cato Client run side by side on the same endpoint. On-site the ZCC tunnel rides inside the Cato socket tunnel; when remote, the ZCC and Cato SDP tunnels co-exist on the workstation. Internet security stays enforced by ZIA the whole time — right up to the moment the converted Cato policy is proven and the cutover flips web egress from ZIA to the Cato PoP.

The one line that beats the objection

“Internet security never goes dark. ZIA keeps protecting web traffic — reached over an interim IPsec bridge from the Cato PoP — until Cato’s converted policy is live and proven. Only then does internet flip to the PoP, and only then does ZCC come off. Every phase is independently reversible.” The full four-phase policy conversion behind that flip is in the Zscaler (ZIA / ZPA) to Cato use case.
